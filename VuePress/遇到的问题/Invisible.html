    # 这里用来记录无法正常显示博文的情况

    # 第一种

    • 第一种,也是最开始遇到的,我的项目实战码匠笔记里第一天的内容无论如何不能显示,经过我整篇博文反复的折腾.
    • 我发现了原来是不能直接在博文中写标签,如果要写也该引用代码的方式来写,比如这样<div>哈哈</div>

    # 第二种

    • 这次我是遇到了一种情况,某个md笔记文件只要放到项目docs文件夹下
    • 并且更新,更新就会失败
    • 不知道具体原因,可能是因为md文件里写的有些句子语法会冲突
    • 于是我开始找原因
    • 注意,找原因的方法就是在本地运行起博客项目,然后去浏览器f12查看报错就可以排查

    【高级】Steins 16:51:37 你这个，着实问到我了，引入项目你要是跟着教程的，那就在项目下命令行npm run dev 【高级】Steins 16:53:49 npm run docs:dev 【高级】Steins 16:56:49 要是没报错就npm run docs:build

    1. 第一个发现的原因
      • 笔记里不能直接写router-link标签,是带标签符号的那种,如果写了
      • 必须要用``符号包裹起来,不然肯定出问题
    2. 第二个发现的问题,笔记里不能写模板语法,也就是两层{}括号这种,然后在里面调用$store是会报错的, 即使包裹在``里也不管用
    3. 第三个问题,就是不能使用双层括号,都会被解析不管是不是被``包裹,除非你用代码块包裹起来 那没问题,否则被包裹的变量,方法都会导致出问题
    4. 最后一个问题也是双层包裹的问题,是id被包裹了起来,这下全都ok
